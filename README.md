## üèóÔ∏è Architecture

This project implements an end-to-end automated testing workflow for a Student Management API:

1. **Fetch All Students** - Initial data retrieval
2. **Create Student** - Dynamic student creation with validation
3. **Verify Addition** - Confirm student exists in database
4. **Update Student** - Modify student information
5. **Verify Update** - Validate changes persisted
6. **Fetch Specific Student** - Retrieve by ID
7. **Add Address** - Create address records
8. **Verify Address** - Confirm address linkage
9. **Delete Student** - Remove student record
10. **Verify Deletion** - Ensure complete removal
